# Constraints

Rules that must be followed. Violating these causes data loss, broken deploys, or silent bugs.

---

## Inviolable

**1. Never change the Luhn algorithm.**
`calculateCheckDigit()` and `isValidIMEI()` in `index.html` implement the standard Luhn algorithm. IMEIs generated by this tool are used with carrier systems — invalid IMEIs cause downstream failures. If you suspect a bug, write a test against known-good IMEIs before touching the code.

**2. Never remove the UNIQUE constraint on `imei_history.imei`.**
Duplicate IMEIs in the history would corrupt the `historySet` deduplication logic and could cause the same IMEI to be assigned to multiple clients. The `INSERT OR IGNORE` pattern depends on this constraint being enforced.

**3. Never mutate `tac-database.json` at runtime.**
It is a static file. Runtime TAC changes go through D1 (`tac_additions`, `tac_removals`) and the `/api/tac` endpoint. Attempting to write to the JSON from a Worker is not possible (it's a static asset) — but do not add any code that tries.

**4. `src/worker.js` routing must stay in sync with `functions/api/*.js` exports.**
If you add a new HTTP method to an API handler (e.g., `onRequestPut` in `history.js`), you must also add the corresponding route in `src/worker.js`. There is no auto-discovery.

---

## High Caution

**5. D1 schema changes require `schema.sql` to be updated AND the change applied to production.**
`schema.sql` is the source of truth. If you alter the schema (add column, rename column, add table), update `schema.sql` first. Then apply to production D1 using `wrangler d1 execute` or the MCP `d1_database_query` tool. The schema is not auto-applied on deploy.

**6. Column names in `schema.sql` must match what API handlers reference.**
The handlers use raw column names in SQL strings (e.g., `has_physical_sim`, `is5g`). If a column is renamed, update every SQL query in `functions/api/` that references it.

**7. The `DB` binding name in `wrangler.jsonc` must match what handlers use.**
All handlers access `env.DB`. If the binding name is changed in `wrangler.jsonc`, it must be changed everywhere in the codebase. Currently it's `DB` — do not rename without a full grep.

**8. `npm run build` must run before `wrangler deploy`.**
`wrangler.jsonc` points `assets.directory` at `./dist`, which doesn't exist until `npm run build` creates it. The `npm run deploy` script chains them correctly. Never run `wrangler deploy` directly in CI — always use `npm run deploy`.

---

## Convention

**9. All API responses must include CORS headers.**
The `CORS` constant at the top of each API handler file must be included in all responses. The frontend may be served from a different origin during local dev.

**10. Client names are stored as-trimmed strings.**
The `/api/clients` POST handler trims and rejects empty names. The frontend also trims. Do not change this behavior — client names in history are matched by exact string.

**11. The `agent/` directory is for the AI agent operating system only.**
Do not put application code, assets, or user-facing documentation in `agent/`. This directory is read by the AI agent at session start. Keep it concise and accurate.
